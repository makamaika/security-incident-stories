# サイバー事件アーカイブ 〜2つの実話で学ぶ情報セキュリティ〜

日本で実際に起きた **2つの情報セキュリティ事件** を、興味のない人でも最後まで読めるように
「体験しながら学べる」Webアプリにまとめた、ビルド不要の静的サイトです。

- **大阪の大病院 ランサムウェア事件**（2022年）… *外からの攻撃*
- **NTT西日本系 個人情報928万件 流出事件**（2013〜2023年）… *内からの不正*

「外からこじ開ける強盗」と「内側の鍵を持った人がこっそり持ち出す泥棒」という
**正反対の2事件**を対比できるのが特徴です。

---

## 4つのモード

| モード | 内容 |
|---|---|
| ① 体験する（紙芝居） | その日に何が起きたかを、イラスト付きの場面で1枚ずつ追う |
| ② あなたならどうする？（判断ゲーム） | 担当者になって全8問を選択。間違えると「現実に起きた被害」が出る |
| ③ 数字で見る | 被害の大きさをカウントアップで体感（73日 / 928万件 …） |
| ④ 2つを比べる | 外部攻撃 vs 内部不正を左右で対比 |

そのほか、難しい用語（ランサムウェア・特権ID など）は本文中の<u>下線つき語</u>を
タップ／クリックすると例えで解説が出ます。右上の「動き ON/OFF」でアニメーションも切替可能。

---

## ローカルで開く

- いちばん簡単：`index.html` をブラウザで開くだけで動きます。
- きちんと確認したいとき（推奨）：このフォルダで簡易サーバーを立てます。
  ```bash
  python -m http.server 8000
  ```
  → ブラウザで `http://localhost:8000/` を開く。

外部ライブラリ・ビルド工程は不要です（フォントのみGoogle Fontsを読み込み）。

---

## GitHub Pages で公開する

1. GitHub で新しいリポジトリを作る（例：`security-incident-stories`）。
2. このフォルダの中身（`index.html` / `css/` / `js/` / `README.md`）を **リポジトリの直下**に置いて push する。
3. リポジトリの **Settings → Pages** を開く。
4. **Build and deployment → Source** を「**Deploy from a branch**」にし、
   Branch を `main`（フォルダは `/root`）に設定して **Save**。
5. 数十秒〜数分後、`https://<あなたのユーザー名>.github.io/security-incident-stories/` で公開されます。

> パスはすべて相対指定（`css/style.css` など）なので、リポジトリ直下に置けばそのまま動きます。

## 発表（スライド）での使い方のヒント

- スライドには各モードのスクリーンショットを貼り、本番でリンクを開いて
  「②あなたならどうする？」を実演すると盛り上がります。
- 「③数字で見る」をスクロールして数字が動く様子は、被害規模のインパクトを伝えやすいです。
- 「④2つを比べる」は、外部攻撃と内部不正の違いをまとめるスライドにそのまま使えます。
